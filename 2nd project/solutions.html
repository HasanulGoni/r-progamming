Exercise 2: Programming

We have to generate three functions: one that randomly chooses three symbols, another one that computes the prize and a third one that allows us to play the slot machine.

2.1

First, the function that chooses the three symbols.

get_symbols <- function() {
  wheel <- c("DD", "7", "BBB", "BB", "B", "C", "0")
  sample(wheel, size = 3, replace = TRUE,
         prob = c(0.03, 0.03, 0.06, 0.1, 0.25, 0.01, 0.52))
}

We can use the function sample(), which randomly selects three symbols of the object wheel. We can specify each probability by using the argument `prob=. In a next step, we compute the prizes, based on the three symbols.

score <- function (symbols) {
  same <- symbols[1] == symbols[2] & symbols[2] == symbols[3]
  if (same) {
    prize <- 100
  } else {
    prize <- 0
  }
  prize
}

If the three symbols are the same, the prize is 100 and else 0. Now we can put these two functions together and create a third function. The function play() randomly selects three symbols and then computes the prize, based on these three symbols.

play <- function() {
  symbols <- get_symbols()
  print(symbols)
  score(symbols)
}

Finally, we can play with the slot machine.

play()

Alternatively, we can also use vectorized code for the function score().

score <- function (symbols) {
  same <- symbols[1] == symbols[2] & symbols[2] == symbols[3]
  ifelse(same, 100, 0) 
}

2.2

Again, we use three functions but we only have to change the function score(). We have to add an additional condition to exclude the three zeros and we need some more else if statements. We also have to define the bars and the cherries.

score <- function (symbols) {
  same <- symbols[1] == symbols[2] & symbols[2] == symbols[3]
  bars <- symbols %in% c("B", "BB", "BBB")
  cherries <- sum(symbols == "C")
  if (same & symbols[1]!="0") {
    prize <- 100
  } else if (all(bars)) {
    prize <- 50
  } else if (cherries>=1) {
    prize <- 10  
  } else {
    prize <- 0
  }
  prize
}

However, we can also use vectorized code to write this function.

score <- function (symbols) {
  same <- symbols[1] == symbols[2] & symbols[2] == symbols[3]
  bars <- symbols %in% c("B", "BB", "BBB")
  cherries <- sum(symbols == "C")
  ifelse (same & symbols[1]!="0", 100, ifelse(all(bars), 50, ifelse(cherries>=1, 10, 0)))
}

We know have to rerun the code for the function play() and then we can play with the new slot machine.

play <- function() {
  symbols <- get_symbols()
  print(symbols)
  score(symbols)
}
play()

Exercise 3: Data Formats

3.1

There are actually two problems, the function xmlTreeParse() does not support links with \(https\) and the linked file is not a XML-file. To overcome these problems we can use the function GET() from the package httr. The function GET() makes a request which saves the response in a string with a read object to it. This allows us to parse it easily with the XML parser.

library(httr)
library(XML)
fileURL <- GET(url = "https://www.parkleitsystem-basel.ch/alte_site/rss_feed.php")
xml_doc <- xmlTreeParse(file = fileURL)

To get the top-level nodes we use the function xmlRoot(). The relevant data is in the first element of the top-level nodes, therefore we simply add [[1]].

items <- xmlRoot(xml_doc)[[1]]

In a next step, we want to extract the relevant data and save it as an R object. For this purpose, we can use a loop. The function xmlValue() extracts the content of these leaf nodes. Before we can run the loop, we first have to create two empty vectors where we can save the extracted data.

location <- vector("character", length(items))
places <- vector("character", length(items))
for (i in seq(6,length(items),1)) {
  name <- xmlValue(items[[i]][[1]])
  location[i-5] <- name
  number <- xmlValue(items[[i]][[3]])
  places[i-5] <- number
}

In a last step we can combine these two vectors with the function cbind() and convert it to a data frame. To make sure that the variables will be of type character, we use the argument stringsAsFactors=F.

parking <- as.data.frame(cbind(location, places), stringsAsFactors = F)

The use of XPath is somewhat less complicated. To be able to use XPath, we have to parse the XML-file with the function xmlParse().

xml_doc <- xmlParse(fileURL)

Now we can use the function xpathSApply() to apply the function xmlValue() on each element of the specified XPath.

location <- xpathSApply(xml_doc, path = "//item/title", xmlValue)
places <- xpathSApply(xml_doc, path = "//item/description", xmlValue)

We can now combine these two vectors again with the function cbind() and convert it to a data frame.

parking <- as.data.frame(cbind(location, places), stringsAsFactors = F)

Additionally, we can extract just the numbers of the free parking places with the function str_extract_all() from the package stringr. We have to define a regular expression that extracts all numeric values. We can do this with the argument pattern=. If we convert the variable to numeric, we can then also compute the sum of all available parking places in Basel.

library(stringr)
parking$places <- as.numeric(str_extract_all(parking$places, pattern = "[0-9]+"))
sum(parking$places)

We can use the function read_html() from the package rvest to read the HTML code and convert it to an R object.

To select the nodes from the HTML document we use the function html_node() and specify the argument xpath=. The XPath is looking for the fourth table from the div elements from elements that have an id attribute value equal to 'mw-content-text'. The expression //* matches any element with id='mw-content-text'. We can then easily convert the output to a table with the function html_table().

tab_node <- 
  html_node(x = mvp, 
            xpath = "//*[@id='mw-content-text']/div/table[4]")
tab <- html_table(x = tab_node)

We can use the same functions when we select the nodes with CSS. All we have to change is the argument css=. The css selector selects the 20th child of the div element in an element that has an id attribute value equal to 'mw-content-text'.

tab_node <-

            css = "#mw-content-text > div > :nth-child(20)")
tab <- html_table(x = tab_node)

4.1

The function summarise() in combination with group_by() allows us to compute the mean, the standard deviation, and the number of observations by origin. To compute the standard error and add it as a new variable to the existing data frame, we can use the function mutate(). The functions arrange() and desc() orders the data in descending order. Here it is helpful to use the pipe.

flights %>% 
  group_by(origin) %>% 
  summarise(delay = mean(dep_delay, na.rm=T),
            obs = n(),
            sd = sd(dep_delay, na.rm=T)) %>%
  mutate(se = sd/sqrt(obs)) %>%
  arrange(desc(delay))

4.2

We can use the same functions as before, we just have to exchange the variables.

flights %>% 
  group_by(carrier, month) %>% 
  summarise(canceled = sum(is.na(dep_delay)),
            total = n()) %>%
  mutate(share_canceled = (canceled/total)*100) %>%
  arrange(desc(share_canceled))

4.3

We select the necessary variables with the function select() and then we can summarise all these variables with the function summarise_all().

d_stats <- flights %>% 
  select(dep_delay, arr_delay, air_time, distance) %>%
  summarise_all(list(Mean=mean, SD=sd, Min=min, Max=max), na.rm=T)

Usually, descriptive statistics have in each row a variable and the columns are the statistical computations. The output now is just all in one row. We can make the data longer with the function pivot_longer(). We have to use three arguments: the columns to be reshaped cols= (here we use a placeholder “.”, which refers to the data d_stats), the name of the column to create names_to= (the \(.value\) sentinel indicates that component of the name defines the name of the column containing the cell values), and where to separate the columns into columns and rows names_pattern=. The last argument uses a regular expression (.*)_([^0-9]+) (this expression matches everything before the last underline, followed by an underline and everything else except numbers). The expression .* is greedy and matches everything until the last underline.

d_stats %>%
  pivot_longer(cols = names(.), names_to = c("Variables", ".value"),
               names_pattern = "(.*)_([^0-9]+)")

Exercise 7: String Manipulation

To read the txt-file we use the function read_file() from the package readr. The package stringr provides many helpful functions for string manipulation.

library(stringr)
library(readr)
latlong_raw <- read_file("data/LatLongFile.txt")

The function str_extract_all() allows us to extract matching patterns. The argument pattern= needs a regular expression. The used regular expression extracts the latitude (we need three backslashes in total because the backslash is here a literal character) with at least one number followed by a point (literal character –> two backslashes) followed by at least one number. The longitude is slightly different because it contains negative values, therefore we add the dot as a metacharacter before the numbers.

lat <- str_extract_all(string = latlong_raw, pattern = "lat=\\\"[0-9]+\\.[0-9]+")
lon <- str_extract_all(string = latlong_raw, pattern = "lon=\\\".[0-9]+\\.[0-9]+")

The output of the function str_extract_all() is a list. We use the function unlist() to convert it to an atomic vector. Then we can create a data frame with the function tibble() form the package tidyverse or we can use the function data.frame().

lat <- unlist(lat)
lon <- unlist(lon)
tibble(lon = lon, lat = lat)

We only want the numbers without anything else. Therefore, we can apply the function str_extract_all() again to extract the numbers. We use the regular expression \([0-9].*[0-9]\) to extract the latitude. This regular expression is looking for a pattern that starts with a number and ends with a number, regardless of what is in between. This is what we call a greedy expression. Alternatively, to extract the longitude, we search for a pattern with at least one negative number, followed by a dot and ends with at least one number. These are two different approaches but both with the same output. Again we use the function tibble() to create a data frame.

lat <- unlist(str_extract_all(string = lat, pattern = "[0-9].*[0-9]"))
lon <- unlist(str_extract_all(string = lon, pattern = "-[0-9]+\\.[0-9]+"))
latlong <- tibble(lon = lon, lat = lat)

Finally, we can convert the two variables to numeric.

latlong <- latlong %>%
  mutate_all(as.numeric)
head(latlong)

murders %>%

tab_merge <- merge(x = murders, y = results_us_election_2016, all.x = T, all.y = T)

Exercise 10: Aggregate Data

10.1

Load the data.

data(CO2)

We first have to generate a function, which computes all the necessary statistics. Then we can use the function aggregate() to compute the statistics by Type and Treatment.

sumfun <- function(x){
  c(n = sum(!is.na(x)), mean = mean(x), sd = sd(x), min = min(x), max = max(x))
} 
aggregate(x = CO2$uptake, by = CO2[c("Type","Treatment")], FUN = sumfun)

An alternative way of how we can use the function aggregate() is the following:

aggregate(uptake~Type+Treatment, CO2, FUN = sumfun)

Alternatively, we can also use the functions group_by() and summarise() from the package tidyverse to solve this exercise.

library(tidyverse)
CO2 %>%
  group_by(Type, Treatment) %>%
  summarise(n = sum(!is.na(uptake)),
            mean = mean(uptake),
            sd = sd(uptake),
            min = min(uptake),
            max = max(uptake))

10.2

We can use a loop for solving this exercise. We loop over all unique values of the variable Plant. This allows us to scale the variable uptake separately for each Plant. For scaling we can simply use the function scale().

plants <- unique(CO2$Plant)
vector <- vector()
for (i in plants){
  vector <- c(vector, scale(x = CO2$uptake[CO2$Plant==i], 
                            center = aggregate(uptake ~ Plant, CO2[CO2$Plant==i,], FUN = mean)[[2]],
                            scale = aggregate(uptake ~ Plant, CO2[CO2$Plant==i,], FUN = sd)[[2]]))
}

Alternatively, the same result can we obtain with the tidyverse package.

CO2 %>% 
  group_by(Plant) %>% 
  mutate(mean=mean(uptake),sd=sd(uptake)) %>% 
  mutate(test=(uptake-mean)/sd)

ggplot(data=iris,

We first have to compute the mean values of these four variables with the function summarise(). Then we have to reshape data with the function pivot_longer() and sort it with the function arrange(). Now we can use the function ggplot() to create the plot with the geom geom_polygon(). The set of aesthetic mapping we are using is group=, color=, y=, and x=. We use the a polar coordinate system for our pie chart coord_polar() and theme_minimal(). We let the names of the axes empty with xlab() and ylab().

               fill = NA) +
